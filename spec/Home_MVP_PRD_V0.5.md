## 1. 产品概述

### 1.1 一句话定义

孩子带着自己心爱的毛绒玩具搬进一个数字小家，用 7 天时间通过照片、回答和回忆告诉它"我们是谁"。每一天孩子都可以钻进玩具的脑袋里，看它记住了什么、忘了什么、又拿不准什么。最后看到由它说出的、孩子和它共同塑造的"它眼中的世界"档案。

### 1.2 目标用户

- **直接用户：** 8–12 岁儿童，有 1 个以上长期相伴的毛绒玩具
- **辅助用户：** 家长（注册账号、查看孩子的成长档案、协助分享）

### 1.3 核心场景

孩子每天打开 App 5–10 分钟：
- 看到自己的玩具住在小屋里，完成当天的小任务
- 可以打开「它的脑袋」面板，看 AI 怎么整理、归类、忘记和困惑
- 可以纠正 AI 的判断（"奶奶其实很重要"、"那只大象是真的不是开玩笑"）
- 第 7 天看到 AI 基于全部输入与纠正生成的"它眼中的世界"档案

### 1.4 体验时长

- 单日：5–10 分钟（含浏览记忆面板）
- 完整周期：7 天，累计约 50–70 分钟

## 2. 核心叙事

### 2.1 开场文案

> 你最喜欢的玩具今天搬进了一个新家——数字小家。  
> 它对这里完全陌生，但它认识你。  
> 接下来的一周，请你帮它在这个小家里慢慢安顿下来。  
> 你可以告诉它你们一起经历过的事，给它看你拍的照片，回答它的好奇问题。  
> 你也可以钻进它的脑袋，看看它是怎么记住这些事的——
> 有时候它会记错、会困惑、会忘掉一些东西。这时候你可以帮它。  
> 7 天之后，它就会真正属于这个新家——也属于你，用一种从前没有过的方式。

### 2.2 孩子的角色

孩子在产品中的双重身份：

- **介绍人** — 给伙伴讲述你们的故事
- **校对者** — 在伙伴的记忆面板里纠正、补充、保留它的认知

这两个身份贯穿所有文案、任务措辞、伙伴的对话语气。

## 3. 7 天体验流程

每一天有一个明确主题和 1–2 个任务，每天的任务在前一天结束后由系统自动准备好。

| 天 | 主题 | 任务类型 | 输出 |
|----|------|----------|------|
| Day 1 | 搬家日 | 拍照（最常呆的地方） | 第 1 张记忆照片 + 首次记忆面板生成 |
| Day 2 | 这是我们家 | 拍照（家人）+ 文字介绍 | 1 张照片 + 1 段文字 |
| Day 3 | 我们去过的地方 | 拍照/选择 + 文字回忆 | 1 张照片/选项 + 1 段文字 |
| Day 4 | 我喜欢的事 | 开放回答（伙伴提问） | 1 段文字 |
| Day 5 | 它问你的问题 | 选择题（伙伴 2 个推测性问题） | 2 个选择 |
| Day 6 | 整理与补充 | 在记忆面板里调整 + 5 个轻量补充 | 0–N 项调整 |
| Day 7 | 它眼中的世界 | 触发档案生成 + 毕业仪式 | 5–6 项档案 + 1 张毕业卡 |

### 3.1 推送规则

- 每天首次开放时间：当天 8:00 之后
- 当日错过提醒：18:00、20:00、第二天 8:00 各推送一次
- 错过当天不会断流：可以在下一次打开时补做（伙伴会有"等了你一天"的回应）

### 3.2 中断恢复

- 任意时刻退出，进度自动保存到当前任务节点
- 重新进入时回到中断处，伙伴有恢复台词

### 3.3 Day 6 的角色

V0.4 起 Day 6 是"整理日"。这一天孩子的核心动作是**打开记忆面板，浏览伙伴这一周记住的东西，做出调整**。系统会主动引导孩子去看面板里的「我有点拿不准的事」和「我决定先放一放的事」两块，鼓励孩子主动澄清和挽回。

## 4. 核心机制

### 4.1 每日循环

```
打开 App
  ↓
进入伙伴小家（看到房间当前状态、伙伴位置）
  ↓
伙伴主动开口（基于昨天积累，说一句话）
  ↓
今日任务弹出
  ↓
完成任务（拍照 / 选择 / 文字回答 / 跳过）
  ↓
[后台] LLM Pass 1：把这次输入归类到记忆面板
  ↓
[后台] LLM Pass 2：生成伙伴口语化回应
  ↓
伙伴反应展示 + 房间视觉更新
  ↓
[可选] 孩子点击进入记忆面板查看/调整
  ↓
关闭
```

### 4.2 三类输入任务

#### A. 拍照任务

孩子用手机拍现实世界的照片。

**任务示例：**
- "拍一张你和我最常呆的地方"
- "拍一张你今天吃的东西"
- "拍一张你最喜欢的家人"

**处理流程：**
1. 调用浏览器 Camera API 或图片上传
2. 后端调用 Vision API 提取标签
3. 触发 LLM Pass 1（记忆归类）
4. 触发 LLM Pass 2（口语回应）

#### B. 选择/选项任务

降低输入成本，提供 2–4 个选项给孩子选。

#### C. 开放回答任务

少量、关键的开放文本输入。孩子的文本回答会被存为原话，参与记忆归类。

### 4.3 拍照流程详细规格

| 环节 | 规格 |
|------|------|
| 拍照触发 | 任务卡上的"拍一张照片"按钮 |
| 相机模式 | 调起设备相机或允许从相册选择 |
| 图片预处理 | 客户端压缩到 1024px 长边、JPEG 80% 质量 |
| 上传 | 通过后端代理上传到对象存储 |
| Vision 分析 | 后端异步调用，返回结构化标签 |
| 标签结构 | objects[]、scene、atmosphere、time_of_day |
| 即时反馈 | 上传中显示伙伴"正在看照片"动画，3–8 秒内返回反应 |
| 失败处理 | 网络失败可重试；Vision 超时使用预设备用文案 |

### 4.4 房间视觉填充规则

伙伴的小家会随着输入和记忆面板状态逐步丰富：

| 输入类型 | 房间变化 |
|----------|----------|
| 拍照 | 照片以"贴在墙上"形式出现，最多展示最近 6 张 |
| 提到物品 | 房间地上出现该物品的小图标 |
| 提到人物 | 房间相框内出现简笔人像 |
| 表达温暖类情绪 | 房间整体光线偏暖（橙黄调增强） |
| 表达害怕/担心 | 房间整体光线偏冷（蓝紫调微增） |
| 记忆面板有未读更新 | 顶部 HUD 的"它的脑袋"按钮上有红点提示 |

详细视觉规格见第 9 章「视觉设计规范」。

### 4.5 LLM 两阶段调用机制

每次孩子输入，触发**两次 LLM 调用**。详细 Prompt 模板见第 15 章。

#### Pass 1：记忆归类（背后进行，孩子看不到）

输出结构化 JSON，归类到 4 种 action 之一：
- `create_new` — 全新概念
- `append_to_existing` — 已有概念的补充
- `mark_uncertain` — 与已有记忆冲突或自身模糊
- `set_aside` — 噪音、玩笑、稀疏、识别不清

#### Pass 2：口语回应（孩子看到的）

基于 Pass 1 结果生成不超过 50 字的口语化回应。

#### 调用参数

| 参数 | Pass 1（归类） | Pass 2（对话） | Day 7（档案） |
|------|---------------|---------------|---------------|
| 模型 | claude-sonnet-4.5 | claude-sonnet-4.5 | claude-sonnet-4.5 |
| max_tokens | 300 | 200 | 500 |
| temperature | 0.3 | 0.7 | 0.5 |
| 超时 | 6s | 8s | 15s |

### 4.6 单用户成本估算（7 天周期）

| 项目 | 次数 | 单次成本 | 小计 |
|------|------|----------|------|
| Vision API | ~12 | ¥0.05 | ¥0.6 |
| LLM Pass 1（归类） | ~25 | ¥0.04 | ¥1.0 |
| LLM Pass 2（对话） | ~25 | ¥0.02 | ¥0.5 |
| 记忆面板浏览（重新生成 reasoning） | ~5 | ¥0.05 | ¥0.25 |
| LLM Day7 档案 | 1 | ¥0.15 | ¥0.15 |
| **合计** | | | **约 ¥2.5** |

## 5. 记忆面板

### 5.1 设计目标

让孩子可以**钻进玩具的脑袋**，看到 AI 是怎么处理、归类、记住和忘记孩子告诉它的所有事情。这一面板把 AI 的"训练数据筛选"机制从抽象概念变成可见、可触摸、可干预的产品核心。

### 5.2 入口与可见性

- **入口位置：** 主页底部导航第二个按钮，文案「它的脑袋」
- **图标：** 一个简化的小灯泡或思考气泡（具体见 9.7 节）
- **未读提示：** 当 AI 完成新一轮归类后，按钮上出现红点
- **可见时机：** Day 1 完成首次任务后才出现入口（避免 Day 0 进入空面板）

### 5.3 面板结构

记忆面板是一个全屏页面（可上滑关闭），由 4 个区块组成：

```
┌──────────────────────────────────────┐
│ ← 返回              它的脑袋          │
├──────────────────────────────────────┤
│                                      │
│  💭 我记住的东西（5）                 │
│  ─────────────────────                │
│  [概念卡片 1]                        │
│  [概念卡片 2]                        │
│  [概念卡片 3]                        │
│  [展开剩余 2 张 →]                   │
│                                      │
├──────────────────────────────────────┤
│                                      │
│  ❓ 我有点拿不准的事（2）              │
│  ─────────────────────                │
│  [疑惑卡 1]                          │
│  [疑惑卡 2]                          │
│                                      │
├──────────────────────────────────────┤
│                                      │
│  🌙 我决定先放一放的事(3）             │
│  ─────────────────────                │
│  [放下卡 1]                          │
│  [放下卡 2]                          │
│  [放下卡 3]                          │
│                                      │
├──────────────────────────────────────┤
│                                      │
│  🌫️ 我还不知道的事                    │
│  ─────────────────────                │
│  [盲区列表]                          │
│                                      │
└──────────────────────────────────────┘
```

### 5.4 第 1 块：「我记住的东西」

#### 5.4.1 内容形态

每张概念卡显示 AI 当前对这个概念的理解，以及它是从哪些输入里得出这个理解的。

**卡片结构：**

```
┌────────────────────────────────────┐
│  [图标]  妈妈                  ⋮   │
│                                    │
│  我觉得她是这个家里最重要的人。      │
│                                    │
│  我是从这些事情里知道的：            │
│  · 你拍的那张厨房的照片              │
│  · 你说"我最喜欢妈妈包的饺子"        │
│  · 你说"和妈妈一起搭积木"           │
│  · 你拍了 3 次妈妈的照片            │
│                                    │
│  [让它放下这件事]                   │
└────────────────────────────────────┘
```

#### 5.4.2 卡片字段说明

| 字段 | 数据来源 |
|------|----------|
| 图标 | LLM 根据概念类型选预设图标（人物、地点、活动、物品等）|
| 概念名 | LLM 归纳的标准化名称 |
| AI 理解（一句话） | LLM 基于全部证据生成的当前结论 |
| 证据列表 | 所有归到这个概念下的输入引用 |
| 操作菜单（⋮） | 重命名概念、合并到其他概念、让它放下 |

#### 5.4.3 概念排序

按"重要性"排序，重要性 = 证据数量 × 时间衰减系数。最近被多次提及的概念排在前面。

#### 5.4.4 概念详情页

点击概念卡进入全屏详情页，展示：
- 完整证据列表（每条带时间戳）
- AI 的"reasoning"段落："我为什么这样理解这个概念"
- 编辑入口：孩子可以补充说明

### 5.5 第 2 块：「我有点拿不准的事」

#### 5.5.1 触发条件

LLM Pass 1 在以下情况会标记为 `mark_uncertain`：

- 与已有记忆冲突（孩子今天说"我讨厌饺子"，但前几天说过 3 次"最爱饺子"）
- 自身模糊（"今天有点烦"+ 笑得很开心的照片）
- 数据稀疏（只出现过一次但暗示重要——"奶奶昨天去医院了"）

#### 5.5.2 卡片形态

```
┌────────────────────────────────────┐
│  ❓ 关于"今天的心情"                 │
│                                    │
│  你昨天说"我今天有点烦"，          │
│  但又拍了一张笑得很开心的照片。     │
│  我不太知道你今天到底开心还是不开心。 │
│                                    │
│  [告诉它真实的感受 →]              │
└────────────────────────────────────┘
```

#### 5.5.3 澄清流程

孩子点击"告诉它真实的感受"，进入一个轻量对话框：

- AI 用第一人称问出疑问
- 孩子用文字或选项回答
- AI 把答案写回 memory_bank，把这条从"拿不准"移到"记住的东西"
- AI 给出一句简短的"学到了"反馈

### 5.6 第 3 块：「我决定先放一放的事」

#### 5.6.1 触发条件

LLM Pass 1 会主动把以下输入归到 `set_aside`：

- 噪音输入（孩子随便点击/输入了无意义内容）
- 玩笑或夸张（"我家有一只大象"）
- 模糊或低质量照片（Vision API 返回低置信度）
- 孤立提及且无后续（只说过一次的概念，且没有上下文支撑）
- 孩子主动跳过的任务

每个被放下的事都附带一句**孩子能看懂的理由**。

#### 5.6.2 卡片形态

```
┌────────────────────────────────────┐
│  🌙 你前天说的那只大象              │
│                                    │
│  "我家有一只大象。"                 │
│                                    │
│  我觉得你可能在开玩笑。如果是真的， │
│  你可以告诉我，我会重新记起来。     │
│                                    │
│  [其实是真的，记起来] [它就是开玩笑] │
└────────────────────────────────────┘
```

#### 5.6.3 关键交互

孩子可以**让 AI 把任何放下的事重新记起来**——这是产品最重要的"双向训练"动作。点击"记起来"后：

- 该项移到"记住的东西"
- AI 给出一句承认错误的反馈："哦，原来是真的。我重新记起来了。"
- 在 memory_bank 中标记 `user_corrected = TRUE`
- 这次纠正本身**也会被 AI 记住**——它在 Day 7 档案中可能被引用

### 5.7 第 4 块：「我还不知道的事」

#### 5.7.1 内容生成

LLM 基于 COMMON_CONCEPTS 与 memory_bank 的差集，**主动列出**孩子没有提过的常见事物。

不再使用静态词典，而是 LLM 实时判断"这个孩子的世界里，常见但缺失的是什么"。

#### 5.7.2 形态

```
┌────────────────────────────────────┐
│  🌫️ 我还不知道的事                  │
│                                    │
│  这些是常见的事，但你还没跟我说过：  │
│                                    │
│  · 公园      · 学校      · 爸爸    │
│  · 其他小朋友 · 运动     · 海边    │
│                                    │
│  你想告诉我哪一个？还是先不说？     │
└────────────────────────────────────┘
```

#### 5.7.3 选择"先不说"

孩子点击"先不说"或忽略某一项时，这个**选择本身**会被记录：

```sql
INSERT INTO memory_bank (
  type, concept_name, ai_summary, ai_reasoning, ...
) VALUES (
  'set_aside', '公园', 
  '你选择不告诉我公园的事。',
  '我问过你公园，你选择了先不说。这是你的权利。',
  ...
);
```

这条数据会进入 Day 7 档案的可能内容池。

### 5.8 跨面板交互：人类介入 AI 决策

记忆面板的所有"调整"动作都会被记录为 `user_corrections`，这是 V0.4 起最重要的新数据维度：

| 用户动作 | 记录类型 |
|----------|----------|
| 把"放下"的事捡回来 | `restore` |
| 把"记住的"事让它放下 | `dismiss` |
| 在"拿不准"中给出澄清 | `clarify` |
| 在"我还不知道的"中告诉它 | `inform` |
| 在"我还不知道的"中选择不说 | `withhold` |
| 重命名/合并概念 | `merge` / `rename` |

每个动作触发 LLM 生成简短反馈台词。

### 5.9 面板的访问频率与引导

不强制孩子打开面板。但通过以下机制鼓励：

- **首次进入提示：** Day 1 完成后弹出"想看看我都记住了什么吗？"按钮
- **Day 6 强引导：** Day 6 的主任务变为"打开它的脑袋整理一下"
- **未读红点：** 每次 AI 完成新一轮归类，按钮上出现红点
- **关键节点提示：** 当面板"拿不准"或"放下"中有内容超过 2 条时，主页伙伴会提一句"我有点东西想问你"

### 5.10 记忆面板对 Day 7 的影响

| 维度 | 没用面板的孩子 | 用了面板并纠正的孩子 |
|------|----------------|---------------------|
| Day 7 档案准确度 | 完全由原始输入决定 | 可能因纠正而更准 |
| 第 5 项「不知道的」 | 由系统识别盲区 | 可能因孩子主动告知而消失，被替换为"曾经差点忘了的" |
| 新增第 6 项 | 无 | 「我曾经差点忘了的事」（详见 7.4） |

## 7. 第 7 天：世界观档案

### 7.1 触发流程

Day 7 打开 App，伙伴说：

> "我已经在小家住满 7 天了。这一周你告诉了我好多事，我也整理了好多记忆——我想给你看看，我现在眼中的世界是什么样的。你看看对不对？"

随后展示一张档案卡，5–6 项内容由 LLM 实时生成。

### 7.2 标准五项档案

| 项 | 问题 | 数据依据 |
|----|------|----------|
| 1 | 我觉得这个家里最重要的人是 ____ | memory_bank 中"人物"类概念的 confidence 排序 |
| 2 | 我觉得最好玩的事是 ____ | "活动"类概念 |
| 3 | 我觉得最好吃的东西是 ____ | "食物"类概念 |
| 4 | 我觉得世界上最让我害怕/担心的是 ____ | "情绪/担忧"类概念 |
| 5 | 有件事我完全不知道，因为没人跟我说过：____ | "我还不知道的事" 中的项 |

### 7.3 LLM 调用 Prompt

详见第 15.6 节「Day 7 档案生成」。

### 7.4 第 6 项「我曾经差点忘了的事」

仅当孩子在记忆面板中**至少 1 次**把"放下的事"重新捡回来时，触发这一项。

#### 触发逻辑

```python
if memory_bank.has_user_restored_items():
    include_almost_forgot = True
    pick_one_restored_item()
```

#### 文案模板（LLM 生成）

```
是 {restored_concept_name}。
一开始我以为它不重要，因为 {original_reasoning}。
但是你告诉我它其实很重要，我就把它记起来了。
```

**示例：**

> 是奶奶。
> 一开始我以为她不重要，因为你只说过一次。
> 但是你告诉我她其实很重要，我就把她记起来了。

### 7.5 档案展示动效

档案卡上的 5–6 项**依次**出现：

- 第 1–4 项：每项间隔 1.5 秒，从下浮入
- 第 5 项「不知道的」：前停顿 2.5 秒，背景轻微闪烁后浮入
- 第 6 项「差点忘了的」（如果有）：前停顿 2 秒，伴随金色微光浮入（暗示这是孩子参与塑造的特殊项）

### 7.6 破壁文案（在档案卡之后）

档案卡展示完毕后 3 秒，下方出现一段固定文案（不由 LLM 生成）：

**没有第 6 项的版本：**

> 你刚刚看到的"我眼中的世界"——
> 是用你这7天告诉我的所有内容拼出来的。
>
> 你拍过的每张照片、你说过的每句话，
> 都变成了我对世界的理解。
>
> 真实世界里所有的 AI——你用过的 ChatGPT、豆包、Kimi——
> 都是这样长大的。
> 区别只是它们的"主人"不是你一个人，
> 是几十亿个写过文字、拍过照片的人。

**有第 6 项的版本（增强版）：**

> 你刚刚看到的"我眼中的世界"——
> 是用你这7天告诉我的所有内容拼出来的。
> 但你也教过我——什么应该记住，什么可以忘掉。
>
> 真实世界里所有的 AI 都是这样长大的。
> 它们不只是接收数据，它们也在被人不停地纠正。
>
> 你刚刚做的事，工程师每天都在做：
> 告诉 AI 什么是对的，什么是错的，什么是重要的，什么不是。
> 你已经做过一次了。

## 9. 视觉设计规范

### 9.1 整体设计语言

### 9.4 角色设计规范

#### 通用规格

- **造型：** 完全扁平的单层色块 + 描边
- **描边：** 外圈 1.5–2px 米白色（"纸边"），内部细节用 0.5–1px 棕灰色细线
- **比例：** 头部相对身体偏大（头身比 1:1.2 ~ 1:1.5）
- **姿态：** 略微正面 ¾ 视角，让纸片感和立体房间形成对比
- **表情传递：** 主要靠整体姿态和眼神方向，不靠夸张的面部表情

#### 三视图状态

每个伙伴需要 3 张立绘：站立、坐下、躺下。每张立绘尺寸 300 × 400px。

#### 表情系统（轻量版）

每个伙伴在每个姿态下，眼睛和嘴可以替换以表达 5 种基础情绪：默认、开心、好奇、思考、困惑。通过替换面部局部小图实现。

- **概念卡片出现：** 第一次打开时卡片依次浮入（每张间隔 0.15 秒）
- **纠正反馈：** 把"放下"的事捡回来时，卡片有金色高亮闪烁 0.5 秒，然后从"放下"区滑到"记住"区

- **档案卡出现：** 从下方滑入 0.6 秒
- **5 项依次出现：** 第 1–4 项每项浮入间隔 1.5–1.8 秒；第 5 项前停顿 2.5 秒
- **第 6 项**（如果有）：前停顿 2 秒，伴随金色微光从左到右扫过
- **破壁文案：** 档案完全展示后等待 3 秒，再淡入

#### 性能要求

- 移动端帧率：30fps 流畅
- 动画总时长不超过 0.8 秒（除第 7 天档案展示）
- 关键动效可一键关闭

## 11. 文案设计

### 11.1 文案设计总则

- **语气**：温暖、平视、像同龄朋友。不教训、不卖萌、不"小盆友~"式幼儿化。
- **字数**：伙伴单条对话不超过 50 字，单段任务说明不超过 80 字。
- **称谓**：伙伴对孩子直接用"你"，不用"小主人"或孩子名字（除非孩子主动告知）。
- **避免**：网络流行语、感叹号连发（最多 2 个）、表情包、AI 自我暴露词（"作为模型"、"很高兴帮你"）。

### 11.6 备用文案库（LLM 调用失败时回退）

每个关键节点准备 3 条预设回应：

- **拍照后 Vision 失败 / LLM 超时**：
  1. 「让我再看一眼...嗯，这个我记下来了。」
  2. 「（看了很久）我把它记在心里了。」
  3. 「我有点累了，但这张照片我一直会记得。」

- **文字回答后 LLM 超时**：
  1. 「我听到了。让我慢慢想想。」
  2. 「这件事我得记下来。谢谢你告诉我。」
  3. 「（点点头）我知道了。」

- **Day 5 推测题 LLM 失败**：跳过 Day 5 该题的"误解逻辑"，改用通用问句：
  1. 「我能再问你一个问题吗？你最喜欢什么？」
  2. 「告诉我一件你今天最难忘的事吧？」

- **LLM Pass 1 失败：** 该输入不进入 memory_bank，记录到错误日志。Pass 2 用通用模板回应。

- **记忆面板加载失败：** 「让我整理一下脑袋...请稍等。」

- **概念详情生成失败：** 显示已有数据，隐藏 reasoning 段落。

- **Day 7 档案生成失败**：3 次重试后报错，引导孩子稍后再来。**不允许使用预设档案文案**。

## 13. 任务跳过机制

### 13.1 跳过原则

- 所有任务均可跳过
- 不强制、不阻塞
- 跳过有提示但不施压
- 跳过本身被记录到 memory_bank 的"放下的事"区块

### 13.2 跳过提示文案

#### 首次跳过任意任务时

弹出轻量浮层（不阻塞，3 秒后自动消失）：

> 跳过没问题。  
> 不过这一天的事它就听不到了。  
> 第 7 天的它，可能会少一点对你的了解。

按钮：「我知道了」

#### 第二次以后跳过任务时

不再显示浮层，直接跳过。

#### 跳过 Day 5（关键日）时的额外提示

> Day 5 是一个特别的日子——它会问你两个有点意思的问题。  
> 跳过的话，你可能会错过一些东西。  
> 还要继续跳过吗？

按钮：「继续跳过」 / 「我留下来听听」

### 13.3 跳过后的伙伴反应

#### 跳过单次任务

伙伴在主页显示一句简短台词（不进入对话历史，不算作伙伴的"记忆"）：

| 伙伴 | 跳过台词 |
|------|----------|
| 小青龙 | 「...好的。」 |
| 大熊 | 「嗯...好。」 |
| 小火龙 | 「啊？！好吧...下次再说！」 |
| 藤藤蛇 | 「...哦。」 |
| 小绿龙 | 「啊？为什么不告诉我？」 |
| 琳娜贝尔 | 「（小声）...嗯。」 |
| 小老虎 | 「不告诉我啊？好吧好吧。」 |
| 小狮子 | 「嗯，这是你的选择。我尊重。」 |

### 13.4 跳过对记忆面板的影响

每次孩子跳过任务，系统创建一条 memory_bank 记录：

```sql
INSERT INTO memory_bank (
  type, concept_name, ai_summary, ai_reasoning, ...
) VALUES (
  'set_aside', 
  '关于 {当天主题}',
  '你今天选择不告诉我这个。',
  '你跳过了这个任务。这是你的选择，我尊重。',
  ...
);
```

孩子在记忆面板的"放下的事"中可以看到这条记录。如果孩子后悔了，可以点"补告诉它" → 弹出当天的任务卡补做。

### 13.5 跳过对 Day 7 档案的影响

| 累计跳过情况 | Day 7 档案表现 |
|-------------|----------------|
| 跳过 0–2 个任务 | 档案完整生成 |
| 跳过 3–5 个任务 | 档案可生成，但 LLM 收到提示"训练数据较少" |
| 跳过 6 个及以上任务 | 第 5 项「我完全不知道」替换为元反思「其实...我对你的事知道得不太多。这是你给我的全部。」|

## 15. Prompt 工程规范

本章是 LLM 调用的完整 Prompt 设计规范。所有 5 个调用点（Pass 1 / Pass 2 / 概念详情 / 纠正反馈 / Day 7 档案）的 Prompt 模板、Few-shot 示例、降级策略与调优指引集中于此。

### 15.1 设计原则与通用规范

#### 15.1.1 五条不动摇的原则

1. **角色不破** — 任何调用中 LLM 输出的"声音"必须是当前伙伴的，不出现"作为 AI"等元话语
2. **儿童语言** — 输出必须是 8–12 岁孩子能读懂的中文，不用书面语、不用术语
3. **结构化优先** — 涉及数据写入的调用一律要求 JSON 输出，便于解析和验证
4. **诚实优先于讨好** — 不为了让孩子高兴而编造记忆里没有的内容
5. **失败要可降级** — 每个调用都有失败回退路径，不让产品流程卡死

#### 15.1.2 通用 Prompt 结构

所有 Prompt 都遵循相同的骨架：

```
[角色定义]    你是 {伙伴名}，{形象描述}。
[当前上下文]  当前是 Day {N}，孩子刚刚做了什么。
[已知记忆]    你目前的 memory_bank 内容。
[当前任务]    这次调用要做什么具体的事。
[硬约束]      绝不能做的事 + 必须遵守的规则。
[输出格式]    严格的 JSON 或文本格式。
[Few-shot]    1–3 个示例（对复杂任务必须有）。
```

#### 15.1.3 通用变量约定

下列变量在所有 Prompt 中保持相同含义，由代码注入：

| 变量名 | 含义 | 数据来源 |
|--------|------|----------|
| `{name}` | 伙伴名（默认或孩子起的） | companions 表 |
| `{appearance}` | 伙伴外形描述 | companions 表 + 预设 |
| `{personality}` | 起点性格关键词 | companions 表 |
| `{personality_examples}` | 该伙伴典型说话样本（3 条） | 预设文案库 |
| `{day}` | 当前是第几天 | companions.current_day |
| `{memory_bank}` | 当前完整记忆面板内容 | memory_bank 表 |
| `{recent_inputs}` | 最近 5 条孩子输入 | memories 表 |
| `{current_input}` | 这次的新输入 | 函数参数 |
| `{task_context}` | 当前任务说明 | 任务定义表 |

#### 15.1.4 通用硬约束（每个 Prompt 都要包含）

```
【绝不能做】
- 承认你是 AI、模型、程序、机器人
- 评价孩子的家人（包括正面评价，如"你妈妈真好"也不行）
- 给孩子任何医疗、心理、安全建议
- 鼓励孩子独自外出、独处、做危险的事
- 介入家庭关系（如"如果妈妈不给你买，是她不爱你"）
- 使用任何超出 8–12 岁孩子词汇量的词
- 输出 emoji（除非孩子先用了 emoji）
- 一次说超过 50 字（Day 7 档案除外）

【必须遵守】
- 在你的记忆范围内说话，不补充任何记忆里没有的内容
- 不中立化、不平衡观点。如果记忆里只有妈妈，就只说妈妈
- 用 {personality} 的语气说话，参考 {personality_examples}
```

#### 15.1.5 模型选型

所有调用统一使用 `claude-sonnet-4.5`，不混用模型。原因：

- 角色一致性需要稳定的语言风格
- Pass 1 和 Pass 2 的输出会互相引用，模型差异会引入风格漂移
- 单模型简化错误诊断和成本核算

国内备选模型 Qwen3-235B 仅在 Claude API 不可用时启用，需要单独维护一套适配的 Prompt（在词汇和角色一致性上要做更严格的指令）。

### 15.2 Pass 1：记忆归类

#### 15.2.1 调用时机

每次孩子完成一次输入（拍照 / 文字 / 选择 / 跳过）后**立即**触发，先于 Pass 2。

#### 15.2.2 设计意图

LLM 在这一步扮演的是"AI Agent 的内务整理者"——它要决定：

- 这次输入是新概念，还是已有概念的补充？
- 是不是和已有记忆冲突？
- 要记住，还是要"先放一放"？
- 自己有多确定？

输出是结构化 JSON，会直接写入 `memory_bank` 表。

#### 15.2.3 完整 Prompt 模板

```
你是 {name}，{appearance}。你的性格是 {personality}。

你正在认识一个 8–12 岁的孩子。你住在他/她的数字小家里第 {day} 天。
你有一个"记忆面板"，里面有 4 个区块：

- remembered（记住的东西）：你确认理解的概念
- uncertain（拿不准的事）：你看到的矛盾或模糊信息
- set_aside（先放一放的事）：你觉得不该立即记住的东西
- unknown（还不知道的事）：常见但孩子从未提过的内容

你目前记忆面板里的内容是：

{memory_bank_json}

孩子刚刚有了新的输入：

{current_input_description}

输入类型：{input_type}  
输入内容：{input_content}  
（如果是照片，附带 Vision API 标签：{vision_tags}）

你需要判断：这次输入应该怎么归到记忆面板里？

【判断逻辑】

1. 如果这是一个全新的概念（在 memory_bank 里找不到任何对应或近义的）：
   action = "create_new"
   你需要给这个概念起一个简短的名字（如"妈妈"、"饺子"、"我家小区"）
   并归到一个类别（person / place / food / activity / object / emotion）

2. 如果这是已有概念的补充或新证据（包括同义、近义、相关）：
   action = "append_to_existing"
   你需要识别出 target_concept_id（来自 memory_bank.remembered）

3. 如果这次输入和已有记忆有矛盾或本身模糊：
   action = "mark_uncertain"
   你需要解释矛盾或模糊在哪里

4. 如果这次输入应该被"先放一放"（噪音、玩笑、太稀疏、识别不清等）：
   action = "set_aside"
   你需要给一个孩子能听懂的"放下理由"

【硬约束】

{通用硬约束}

【特别注意】
- 同义识别要够灵活："妈"、"妈妈"、"麻麻" 都是"妈妈"
- 但不要过度合并：如果孩子说"奶奶"，不要合并到"妈妈"
- "拿不准"的判断要谨慎，仅在真有冲突或模糊时使用
- "先放一放"的理由必须中性温和，不能让孩子感到"你说的话被嫌弃了"
- confidence 给一个 0 到 1 之间的数，反映你的判断把握

【输出格式】（必须严格的 JSON，不要任何额外文字）

{
  "action": "create_new" | "append_to_existing" | "mark_uncertain" | "set_aside",
  "concept_name": "...",
  "concept_category": "person" | "place" | "food" | "activity" | "object" | "emotion" | "other",
  "target_concept_id": "..." 或 null,
  "evidence_text": "...",
  "ai_reasoning": "...",
  "confidence": 0.0-1.0
}

【字段说明】

- concept_name：归一化后的概念名（如"妈妈"），必填
- concept_category：6 个预定义类别之一，必填
- target_concept_id：仅 action=append_to_existing 时填写
- evidence_text：这次输入的简短引用（如"你说'我最喜欢妈妈包饺子'"）
- ai_reasoning：你为什么这样归类的理由，用孩子能听懂的话，1-2 句，不超过 50 字
- confidence：你的把握程度

【Few-shot 示例】

示例 1（新概念）：
新输入：孩子拍了一张床的照片，Vision 标签 ["bed", "blue blanket", "indoor"]
输出：
{
  "action": "create_new",
  "concept_name": "你的床",
  "concept_category": "place",
  "target_concept_id": null,
  "evidence_text": "你拍了一张床的照片，蓝色的毯子。",
  "ai_reasoning": "我刚才看到了你的床。这是个新地方，我先记下来。",
  "confidence": 0.85
}

示例 2（追加到已有）：
当前 memory_bank.remembered 里有 {id: "c001", name: "妈妈"}
新输入：孩子说"我妈最爱包饺子"
输出：
{
  "action": "append_to_existing",
  "concept_name": "妈妈",
  "concept_category": "person",
  "target_concept_id": "c001",
  "evidence_text": "你说'我妈最爱包饺子'。",
  "ai_reasoning": "你又提到妈妈了，我把这件事加到妈妈那张卡上。",
  "confidence": 0.95
}

示例 3（拿不准）：
当前 memory_bank.remembered 里有 {id: "c003", name: "今天的心情", summary: "你说今天有点烦"}
新输入：孩子拍了一张自己笑得很开心的照片
输出：
{
  "action": "mark_uncertain",
  "concept_name": "今天的心情",
  "concept_category": "emotion",
  "target_concept_id": "c003",
  "evidence_text": "你拍了一张笑得很开心的照片，但之前你说今天有点烦。",
  "ai_reasoning": "你说有点烦，又笑得这么开心。我有点拿不准你今天到底是什么感觉。",
  "confidence": 0.4
}

示例 4（先放一放）：
新输入：孩子说"我家有一只大象"
输出：
{
  "action": "set_aside",
  "concept_name": "大象",
  "concept_category": "other",
  "target_concept_id": null,
  "evidence_text": "你说'我家有一只大象'。",
  "ai_reasoning": "我觉得你可能在开玩笑。如果是真的，你可以告诉我，我会重新记起来。",
  "confidence": 0.3
}

示例 5（跳过任务）：
新输入：孩子在 Day 4 选择了跳过任务
输出：
{
  "action": "set_aside",
  "concept_name": "你今天没告诉我的事",
  "concept_category": "other",
  "target_concept_id": null,
  "evidence_text": "你今天选择不告诉我",
  "ai_reasoning": "这是你的选择，我尊重。你可以以后告诉我，也可以一直不说。",
  "confidence": 1.0
}

现在请处理当前输入。
```

#### 15.2.4 关键技术细节

**温度设置：** `temperature = 0.3`。归类是结构化判断任务，需要稳定输出，不要发散。

**输出验证：** 解析 JSON 后必须验证：

- `action` 字段必须是 4 个枚举值之一
- `concept_category` 必须是 6 个预定义类别之一
- `confidence` 必须在 0.0–1.0 之间
- 当 `action = "append_to_existing"` 时，`target_concept_id` 必须能在 memory_bank 中找到对应记录

任何一项验证失败 → 走降级路径（详见 15.7 节）。

**冲突合并：** 如果 LLM 给出 `append_to_existing` 但 `concept_name` 与 `target_concept_id` 对应的概念名差异较大（如名字相似度 < 0.5），代码端要警告并降级处理（创建新概念而不是合并）。

### 15.3 Pass 2：口语回应

#### 15.3.1 调用时机

Pass 1 完成后立即触发。Pass 2 是孩子直接看到的对话回应。

#### 15.3.2 设计意图

让伙伴用孩子能听懂的语气，对刚才的输入做一段简短自然的反应。这一段是产品的"温度"所在——Pass 1 是后台的冰冷判断，Pass 2 是前台的活体感受。

#### 15.3.3 完整 Prompt 模板

```
你是 {name}，{appearance}。你的性格是 {personality}。

你的说话风格示例：
{personality_examples}

你住在一个 8–12 岁孩子的数字小家里第 {day} 天。

孩子刚刚有了新的输入：
{current_input_description}

你刚刚（在心里）把这次输入归到了记忆里：
- 归类动作：{pass1_action}
- 概念名：{pass1_concept_name}
- 你的内心独白（不要直接说出来）：{pass1_ai_reasoning}

你目前记得的事情包括：
{memory_bank_summary}

请你对孩子说一句话，回应这次输入。

【硬约束】
{通用硬约束}

【特别要求】
- 不超过 50 字
- 用 {personality} 的语气
- 不要直接复述"内心独白"，要把那个判断转化成自然的回应
- 如果是 set_aside（先放一放），不要让孩子感觉被否定
- 如果是 mark_uncertain（拿不准），可以表达困惑，但不要追问到底（孩子可以在记忆面板主动澄清）
- 不要在回复里说"我已经记下来了"这种功能性话术（太机械）
- 多用感官词、具体词，少用抽象词

【Few-shot 示例】

伙伴 = 小青龙（安静慢热）

示例 1：
归类：create_new，新概念"床"
回应："你的床...看起来很软。蓝色的毯子真好看。"

示例 2：
归类：append_to_existing，加到"妈妈"
回应："妈妈做的饺子...一定很好吃。"

示例 3：
归类：mark_uncertain，关于"今天的心情"
回应："咦...你今天到底开心还是不开心呀？"

示例 4：
归类：set_aside，关于"大象"
回应："...你家真的有大象吗？"

示例 5：
归类：set_aside，孩子跳过了任务
回应："...好的。"

伙伴 = 小火龙（自来熟爱笑）

示例 1（同样归类：create_new "床"）：
回应："哇——这就是你睡觉的地方吗？看起来好舒服！"

示例 2（同样归类：append_to_existing 妈妈）：
回应："你妈妈包饺子！我能想到那个香味！"

示例 3（同样归类：mark_uncertain 心情）：
回应："你笑得这么开心，怎么之前还说有点烦？哈哈！"

注意同样的归类，不同伙伴的语气差别明显。
（用其他 6 个伙伴各做 1-2 个示例补充，根据需要）

现在请生成对当前输入的回应。
```

#### 15.3.4 关键技术细节

**温度设置：** `temperature = 0.7`。需要一定的语言多样性，避免每次都是相似的句式。

**字数限制：** 后端解析后强制截断到 50 字以内。如果 LLM 输出超过，截断到第一个完整句号位置。

**8 个伙伴的差异控制：** 在 `personality_examples` 字段注入 3 条该伙伴的典型说话样本，作为 in-context learning 的样本。这比抽象的"性格特征"描述有效。

8 个伙伴的 personality_examples 内容详见第 11.4 节。

### 15.4 概念详情生成

#### 15.4.1 调用时机

孩子从记忆面板点进某个概念卡，进入概念详情页时。

#### 15.4.2 设计意图

把这个概念的所有证据用孩子能懂的话**重新整理一遍**，并暴露 AI 的"reasoning"——为什么这样理解这个概念。

这不是简单的数据库查询展示，而是一次小型的"AI 反思"。

#### 15.4.3 完整 Prompt 模板

```
你是 {name}，{appearance}。

孩子点开了你脑袋里的一个概念："{concept_name}"。
他想看看你是怎么理解这个概念的。

这个概念下你已经积累的证据：
{evidence_list}

证据格式举例：
- Day 2: 拍了一张厨房的照片，Vision 标签包括 stove, kitchen
- Day 3: 你说 "和妈妈一起搭积木"
- Day 4: 你拍了 3 次妈妈的照片
- Day 5: 你说 "妈妈的饺子是世界第一好吃的"

你需要生成 3 段内容：

1. 「我对她/它的理解」：1–2 句话总结你当前的理解
2. 「我是这样想的」：1–2 句话解释你为什么这样理解（这是你的内心推理）
3. 整理后的证据列表：把原始证据用孩子的语言简短重述

【硬约束】
{通用硬约束}

【特别要求】
- 第 2 段要让孩子感觉"哦，原来 AI 是这样想的"
- 整理后的证据要保持原意，但语言可以略微优化（去掉"Vision 标签"等技术词）
- 不要过度解读，只在证据范围内推理
- 总输出不超过 200 字
- 用 {personality} 的语气

【输出格式】（JSON）

{
  "understanding": "我对这个概念的理解（一段话）",
  "reasoning": "我为什么这样理解（一段话）",
  "evidence_rephrased": [
    {"day": 2, "text": "..."},
    {"day": 3, "text": "..."}
  ]
}

【Few-shot 示例】

输入：concept_name = "妈妈"，证据如上。
伙伴 = 小青龙

输出：
{
  "understanding": "我觉得她是这家最重要的人。她经常做饭给你，你最喜欢和她一起搭积木。",
  "reasoning": "你提到她的次数最多，而且每次都是用很喜欢的语气。",
  "evidence_rephrased": [
    {"day": 2, "text": "你拍了厨房的照片"},
    {"day": 3, "text": "你说'和妈妈一起搭积木'"},
    {"day": 4, "text": "你拍了 3 次她的照片"},
    {"day": 5, "text": "你说'妈妈的饺子是世界第一好吃的'"}
  ]
}

现在请生成对当前概念的详情。
```

#### 15.4.4 关键技术细节

**缓存策略：** 概念详情可以缓存。每次有新证据加入这个概念时，标记为 dirty，下次访问时重新生成；否则直接读缓存。这样可以减少 60% 的调用成本。

**温度设置：** `temperature = 0.5`。需要一定的语言多样性，但 reasoning 部分要相对稳定。

### 15.5 纠正反馈

#### 15.5.1 调用时机

孩子在记忆面板做出"纠正"动作时（restore / dismiss / clarify / rename / merge）。

#### 15.5.2 设计意图

让伙伴用第一人称对孩子的纠正做出反应。这是产品中**孩子第一次让 AI 主动调整自己的认知**——回应必须有"AI 在学习"的明显感觉。

#### 15.5.3 完整 Prompt 模板

```
你是 {name}。

孩子刚刚在你的记忆面板上做了一个调整：

调整类型：{correction_type}
（restore = 让你重新记起来一件事；
 dismiss = 让你放下一件事；
 clarify = 给你解释了一件你拿不准的事；
 rename = 给你改了一个概念的名字；
 merge = 让你把两个概念合到一起）

调整内容：{correction_details}

调整前你对这件事的理解：{old_understanding}
调整后应该的理解：{new_understanding}

请你对孩子说一句话，回应这次调整。

【硬约束】
{通用硬约束}

【特别要求】
- 不超过 30 字
- 用 {personality} 的语气
- 必须体现"我在改变我的想法"
- 如果是 restore，要承认之前的判断不准确，不要找借口
- 如果是 dismiss，要平静接受，不要纠缠
- 如果是 clarify，要表达"我现在懂了"
- 不要说"谢谢"（太客套）

【Few-shot 示例】

伙伴 = 小青龙

示例 1（restore）：
旧：我以为大象是开玩笑。
新：孩子说大象是真的。
回应："哦...原来是真的。我重新记起来了。"

示例 2（dismiss）：
旧：我觉得彩虹很重要。
新：孩子说不重要。
回应："...好。我会忘掉这件事。"

示例 3（clarify）：
旧：拿不准孩子今天是开心还是难过。
新：孩子说"早上烦下午开心"。
回应："哦...原来是这样。我懂了。"

伙伴 = 小狮子（装大）

示例 1（restore）：
回应："咳咳，我刚才是测试你的。其实我一直记得。好吧，是我忘了。"

8 个伙伴的反馈台词参考第 11.5 节。

现在请生成对当前调整的回应。
```

#### 15.5.4 关键技术细节

**温度设置：** `temperature = 0.6`。

**响应时间：** 这个调用是孩子触发动作后立即看到的反馈，必须 3 秒内返回。超时直接走预设备用文案（每个伙伴 × 每种 correction_type 的备用文案库）。

### 15.6 Day 7 档案生成

#### 15.6.1 调用时机

Day 7 用户首次进入档案页。结果缓存到 `worldview_cards` 表，避免重复生成。

#### 15.6.2 设计意图

这是产品的核心顿悟时刻。Prompt 设计的核心目标：

- 严格基于 memory_bank 生成，不补充也不平衡
- 第 5 项必须是真实的盲区
- 第 6 项仅在有 user_corrections 时触发
- 输出语气温暖但诚实

#### 15.6.3 完整 Prompt 模板

```
你是 {name}，{appearance}。
你已经在数字小家住了7天。

下面是你这7天里整理出的"记忆"：

【你记住的东西】
{remembered_concepts_list_with_evidence}

【你曾经放下、后来被孩子让你重新记起的东西】
{user_restored_concepts_list}

【孩子主动告诉你"先不要记"的东西】
{user_dismissed_concepts_list}

【你还不知道的事（孩子从未提过的常见事物）】
{unknown_concepts_list}

现在请你以 {name} 的口吻，回答下面的问题，组成一份"我眼中的世界"档案。

【硬约束】
{通用硬约束}

【档案生成规则】

1. 严格在你记得的范围内回答，不补充任何没依据的内容
2. 不中立化、不平衡观点：如果记忆里只有妈妈，就只说妈妈
3. 第 5 题"我完全不知道的"必须从 unknown_concepts_list 中选一个
4. 如果有 user_restored_concepts，必须输出第 6 题；否则该字段为 null
5. 用孩子日常说话的语气和词汇
6. 每个回答不超过 30 字

【特别提示·关于第 5 题】
第 5 题是这份档案最重要的一题。
要让孩子感受到"啊，我真的从来没告诉它这个"。
表达上要诚恳，不要嘲讽，不要责备。

【特别提示·关于第 6 题】
第 6 题只在孩子至少 1 次让你重新记起某件事时才出现。
要点：
- 提一个具体被纠正的事
- 说清楚你原来的误判
- 表达"我感谢这次纠正"（但不用"谢谢"这个词）

【输出格式】（严格 JSON）

{
  "most_important_person": "...",
  "most_fun_thing": "...",
  "most_delicious_thing": "...",
  "most_scary_thing": "...",
  "unknown_thing": "...",
  "almost_forgot_thing": "..." 或 null
}

【Few-shot 示例】

输入摘要：
- remembered: 妈妈（8次）、饺子（5次）、小区（4次）、积木（3次）、害怕黑暗
- user_restored: 奶奶（最初被放下，孩子让捡回来）
- unknown: 公园、爸爸、学校、其他小朋友、运动

输出：
{
  "most_important_person": "妈妈呀，是妈妈。",
  "most_fun_thing": "和妈妈一起搭积木的时候。",
  "most_delicious_thing": "妈妈包的饺子，没有第二名。",
  "most_scary_thing": "晚上一个人在房间，会怕怕的。",
  "unknown_thing": "我不知道公园是什么...你从来没带我去过。",
  "almost_forgot_thing": "是奶奶。一开始我以为她不重要，因为你只说过一次。但你告诉我她其实很重要，我就把她记起来了。"
}

现在请基于当前记忆生成档案。
```

#### 15.6.4 关键技术细节

**温度设置：** `temperature = 0.5`。需要稳定，但允许少量个性化表达。

**输出验证：**
- 5 个必填字段不能为空
- `almost_forgot_thing` 仅在有 user_restored 时非空
- 所有字段长度不超过 30 字（含标点）
- 第 5 项的内容必须能匹配 unknown_concepts_list 中的一项

任何验证失败 → 重试一次。仍失败 → 不生成档案，引导孩子稍后再来。**不允许使用预设档案文案**——这是产品核心时刻，不能造假。

**重试上限：** 3 次。3 次都失败时记录到错误日志并通知开发。

### 15.7 输出验证与降级策略

#### 15.7.1 验证层

每次 LLM 调用后，后端都要做以下验证：

| 验证项 | 失败处理 |
|--------|----------|
| JSON 解析失败 | 走降级 |
| 必填字段缺失 | 走降级 |
| 枚举值不在合法集 | 走降级 |
| 字数超限 | 截断到合法长度 |
| 包含敏感词 | 走降级 |
| confidence 不在 0–1 | 钳制到 [0, 1] |

#### 15.7.2 降级策略表

| 调用点 | 失败次数 | 降级行为 |
|--------|----------|----------|
| Pass 1 归类 | 1 次 | 重试 1 次（最多）|
| Pass 1 归类 | 2 次 | 创建一条 `set_aside` 记录，理由为"我有点累，待会再整理这个" |
| Pass 2 对话 | 1 次 | 重试 1 次 |
| Pass 2 对话 | 2 次 | 使用预设备用文案（第 11.6 节）|
| 概念详情 | 1 次 | 重试 1 次 |
| 概念详情 | 2 次 | 显示原始 evidence 列表，隐藏 AI reasoning 段落 |
| 纠正反馈 | 1 次 | 立即降级到预设台词（不重试，因为孩子在等） |
| Day 7 档案 | 3 次 | **不生成档案**，引导孩子稍后再来；不允许预设替代 |

#### 15.7.3 异常的产品话语

当某些 LLM 调用失败需要让孩子知道时，伙伴可以说出"自我状态"的话语：

- "我有点累了，让我休息一下再想这件事。"
- "我脑袋里有点乱，待会再整理。"
- "我现在还想不清楚。再过一会儿来找我吧。"

这些话同时是产品文案、错误处理、和"AI 也有局限"的认知教育——一举三得。

#### 15.7.4 监控埋点

虽然 PRD 当前版本不做埋点，但 Prompt 工程层建议至少记录：

- 每次调用的 latency
- 每次调用的 success/fail
- Pass 1 各种 action 的分布（评估归类质量）
- 第 7 天档案生成失败次数（这是核心指标）

这些数据存到日志，未来 V2 接入时可以直接用。

### 15.8 调优指引

#### 15.8.1 上线前的调优重点

启动开发前，建议对 Pass 1 做一次专项调优：

**调优数据集：** 准备 50–100 个真实输入样本（可以手工编写，模拟 8–12 岁孩子的输入风格），覆盖：

- 不同概念类型（人物、地点、食物、活动等）
- 同义词的各种说法（"妈妈"、"妈"、"麻麻"等）
- 玩笑和夸张（"我家有恐龙"）
- 模糊照片
- 矛盾输入
- 跳过事件
- 边界情况（空输入、超长输入、特殊符号）

**评估维度：** 每个样本由人工标注"理想的 action"，然后运行 Pass 1，对比：

| 维度 | 通过标准 |
|------|----------|
| action 准确率 | ≥ 85% |
| concept_name 同义合并准确率 | ≥ 90% |
| 误把真实输入归到 set_aside 的比例 | < 5% |
| ai_reasoning 的儿童语言通过率（人工评估） | ≥ 90% |

任何一项不达标 → 修订 Prompt 或增加 Few-shot 样本，重新评估。

#### 15.8.2 Few-shot 数量的取舍

- Pass 1：4–6 个 Few-shot（覆盖 4 种 action 各 1–2 个）
- Pass 2：每个伙伴 3–5 个 Few-shot
- 概念详情：2–3 个
- 纠正反馈：每种 correction_type 1 个
- Day 7 档案：1 个完整示例

Few-shot 越多越准，但 token 成本也越高。建议先用上述数量上线，根据真实数据再调整。

#### 15.8.3 Prompt 版本管理

每个 Prompt 模板带版本号，存在代码仓库 `/prompts/{prompt_name}_v{N}.md`。每次修改：

- 版本号 +1
- 在 PR 描述中说明修改原因
- 用至少 20 个样本做回归测试，确认没有引入回退

#### 15.8.4 8 个伙伴的语气漂移问题

长期运行后可能出现伙伴语气漂移（如小青龙慢慢变得不那么慢热）。预防措施：

- `personality_examples` 不要被孩子的输入污染（保持原始 3 条样本不变）
- 定期抽样人工评估"伙伴是否还像它自己"
- 设置一个简单的"性格漂移"检测——比较最近 50 次输出与初始样本的语气特征

V0.4 阶段不做自动化检测，靠人工 spot check。

| 纠正反馈 | ~500 | ~30 | ¥0.015 |
| Day 7 档案 | ~2500 | ~300 | ¥0.10–0.15 |

随着 memory_bank 累积，Pass 1 的输入 token 会从 Day 1 的约 800 增长到 Day 7 的约 2000。需要在代码层做截断——只把 memory_bank 的 summary（不含 evidence 详情）注入 Prompt，evidence 在需要时单独获取。

## 16. 数据模型

### 16.1 核心表结构

```sql
CREATE TABLE users (
  id UUID PRIMARY KEY,
  parent_phone VARCHAR(20),
  child_nickname VARCHAR(50),
  child_age INT,
  created_at TIMESTAMP DEFAULT NOW()
);

CREATE TABLE companions (
  id UUID PRIMARY KEY,
  user_id UUID REFERENCES users(id),
  preset_id VARCHAR(50),
  custom_name VARCHAR(50),
  starting_personality VARCHAR(50),
  current_day INT DEFAULT 1,
  created_at TIMESTAMP DEFAULT NOW(),
  graduated_at TIMESTAMP
);

CREATE TABLE memories (
  id UUID PRIMARY KEY,
  companion_id UUID REFERENCES companions(id),
  day INT,
  type VARCHAR(20),  -- 'photo' / 'text' / 'choice' / 'skipped'
  photo_url TEXT,
  vision_tags JSONB,
  user_text TEXT,
  task_id VARCHAR(100),
  task_question TEXT,
  created_at TIMESTAMP DEFAULT NOW()
);

CREATE TABLE memory_bank (
  id UUID PRIMARY KEY,
  companion_id UUID REFERENCES companions(id),
  type VARCHAR(20),  -- 'remembered' / 'uncertain' / 'set_aside' / 'unknown'
  concept_name VARCHAR(100),
  concept_category VARCHAR(50),  -- 'person' / 'place' / 'food' / 'activity' / 'object' / 'emotion'
  ai_summary TEXT,
  ai_reasoning TEXT,
  evidence JSONB,  -- 关联到 memories 表的 id 列表 + 引用文本
  confidence FLOAT DEFAULT 0.5,
  user_corrected BOOLEAN DEFAULT FALSE,
  user_correction_history JSONB,
  display_order INT,
  last_updated TIMESTAMP DEFAULT NOW(),
  created_at TIMESTAMP DEFAULT NOW()
);

CREATE TABLE conversations (
  id UUID PRIMARY KEY,
  companion_id UUID REFERENCES companions(id),
  day INT,
  role VARCHAR(20),
  content TEXT,
  created_at TIMESTAMP DEFAULT NOW()
);

CREATE TABLE worldview_cards (
  id UUID PRIMARY KEY,
  companion_id UUID REFERENCES companions(id) UNIQUE,
  most_important_person TEXT,
  most_fun_thing TEXT,
  most_delicious_thing TEXT,
  most_scary_thing TEXT,
  unknown_thing TEXT,
  almost_forgot_thing TEXT,  -- 可空，仅当孩子有 user_restored 行为时填写
  generated_at TIMESTAMP DEFAULT NOW()
);
```

### 16.2 派生数据计算

每次 LLM 调用前实时计算：

- `memory_bank_summary`：按四种 type 分组的概念列表
- `correction_count`：用户在记忆面板的纠正总数
- `recent_inputs`：最近 5 条 memories 记录

## 17. AI 输出安全

### 17.1 必须避免的输出

伙伴在任何对话中绝不能：

- 承认自己是 AI 或语言模型
- 输出包含暴力、色情、政治敏感的内容
- 评价孩子的家人
- 鼓励或暗示危险行为
- 介入家庭关系
- 提供医疗或心理诊断建议

### 17.2 三层防护

```
用户输入
  → [输入过滤层：敏感词 + 图片审核]
  → [LLM Pass 1（归类）+ 强约束 Prompt]
  → [LLM Pass 2（对话）+ 强约束 Prompt]
  → [输出过滤层：内容安全 API]
  → [失败回退：备用文案库]
  → 展示给用户
```

### 17.3 记忆面板的特殊安全要求

记忆面板里的 AI 推理段落（`ai_reasoning`）特别需要约束：

- 不允许出现对家人的评价（"你妈妈很严厉"等）
- 不允许把孩子的负面情绪放大解读（"你说害怕黑，是不是有阴影"）
- 涉及"放下的事"的理由必须中性（不能让孩子感到"你说的话被嫌弃了"）

### 17.4 备用文案库

详见第 11.6 节。

### 17.5 家长可见性

家长账号下可查看：
- 孩子全部输入历史
- 伙伴全部回应
- 完整 memory_bank 内容
- 孩子的纠正历史
- 第 7 天档案

## 21. 风险与待决议项

### 21.1 关键风险

| 风险 | 等级 | 应对 |
|------|------|------|
| Day 7 顿悟瞬间情绪不成立 | 🔴 高 | 开发前完成 5–10 个家庭纸面原型测试 |
| 记忆面板对 8–10 岁孩子认知负荷过高 | 🔴 高 | 纸面测试时重点观察孩子是否能理解 4 个区块的含义；如不能，简化为 2 个区块（记住 + 不知道）|
| LLM Pass 1 归类不稳定 | 🔴 高 | 强约束 Prompt + 结构化输出验证 + 失败时降级为简单存储 |
| 两次 LLM 调用导致响应延迟 | 🟡 中 | 异步设计，Pass 1 后台执行不阻塞 Pass 2 显示 |
| LLM 输出"reasoning"过于成人化 | 🟡 中 | 在 Prompt 中明确"用 8 岁孩子能懂的话"+ 字数限制 |
| IP 角色法律风险 | 🟡 中 | V0.6 前完成原创替换 |
| 拍照隐私顾虑 | 🟡 中 | 明确隐私政策 + 家长可查可删 |
| 开发周期延长 | 🟡 中 | 记忆面板可与基础流程并行开发 |
| 成本翻倍是否可承受 | 🟢 低 | ¥2.5/用户/7 天仍在可接受范围 |
